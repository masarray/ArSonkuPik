# Repository Governance

This repository is prepared for public release and external review.

## Branch model

- `main` should always be releasable.
- Feature work should happen in pull requests.
- Release assets should be generated from protected tags.

## Required checks

Recommended branch protection checks:

- `Validate / Extension, site, and repository checks`
- `CodeQL / Analyze JavaScript`

## Release policy

- Releases are tag-driven.
- Chrome Web Store ZIPs are generated by CI and attached to GitHub Releases.
- Chrome Web Store submission remains manual.
- CRX release is intentionally disabled.

## Security posture

- No private keys in Git.
- No Chrome Web Store API tokens in GitHub Actions.
- Least-privilege workflow permissions.
- Public Pages site for privacy, support, docs, and wiki.
