# Security Policy

## Supported versions

Security fixes are prepared for the latest public release line.

| Version | Supported |
| --- | --- |
| 0.3.x | Yes |

## Report a vulnerability

Please report suspected vulnerabilities privately through the support contact that will be published on the project support page.

Include:

- affected version;
- browser and operating system;
- clear reproduction steps;
- expected impact;
- whether the issue requires user interaction.

Please do not include copyrighted audio, private browsing data, passwords, cookies, tokens, or personal media in reports.

## Scope

In scope:

- extension privilege misuse;
- accidental data collection or disclosure;
- unsafe permission expansion;
- remote executable code introduction;
- audio routing behavior that leaks audio outside the local browser/device path;
- release workflow or package integrity issues.

Out of scope:

- requests to add CRX distribution;
- requests to auto-submit to Chrome Web Store;
- issues caused by modified third-party builds not generated by this repository.

## Release safety

- Web Store packages are ZIP files generated from `extension/`.
- CRX packaging is intentionally disabled.
- Chrome Web Store submission is manual.
- Private keys and Web Store API credentials must never be committed.
- Release artifacts include SHA-256 checksums.
